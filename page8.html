시스템 프롬프트 System Prompt

게임 월드 생성 시 고려해야 할 요소를 시나리오 생성 과정에 반영하기 위해, 선행 연구에서 사용된 프롬프트에 시스템 프롬프트를 추가로 도입하였다. 시스템 프롬프트는 사용자 요청과 LLM 역할 설정으로 구성되며, 이를 통해 리소스 제약이 반영된 Mock 시나리오를 생성하여 파이프라인의 입력으로 사용하였다.

◈ 시스템 프롬프트 구조
A 사용자 요청
시나리오의 내용과 분위기를 결정하는 입력 요소
이야기 구조
장르 플롯
→ 시나리오의 이야기 흐름에 영향
시각 요소
배경 리소스 목록
→ 시나리오의 전체적인 분위기에 영향
구현 사항
게임 규칙 제약 사항
→ 캐릭터 행동, 장소·시간 등 전체에 영향
B LLM 역할 설정
시나리오 품질과 요청 반영을 강제하는 설정
역할 부여
시나리오 작가 역할을 부여하여 대본의 품질을 고려하도록 강제
강제성 부여
'투자자의 요청을 반영하지 않으면 투자를 받을 수 없다'는 조건을 제시하여 사용자 요청을 최우선으로 반영
◈ 입력 항목별 설정 내용
입력 항목 설정 내용
플롯 Quest — 주인공이 특정 목표를 찾아 나서며, 단서를 수집하고 다양한 인물을 만나는 탐색 구조
장르 Light-hearted Comedy — 대화를 통해 오해를 풀어가는 가벼운 판타지 코미디
배경 중세 마을 — 석조 건물과 교회 첨탑이 있는 언덕 아래의 평화로운 마을
게임 규칙 NPC 순차 대화 기반의 선형 진행 — 전투 및 연출 없음
제약 사항 밤낮 변환 배제(오전~저녁 이전) · 실내 표현 배제(모든 장면 야외) · 맵 개수: 1개
