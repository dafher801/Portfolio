자기 피드백 Self-Reflection

자기 피드백 단계에서는 Self-Reflection 기법을 적용하였는데, 이는 LLM이 자신의 사고 과정을 되돌아보도록 유도하여 오류를 식별하고 개선 방향을 스스로 생성하게 하는 기법이다. 본 실험에서는 Self-Reflection 횟수에 따른 게임 월드 생성 품질을 평가하기 위해, 각 횟수 조건(0회, 1회, 5회, 10회)마다 10개의 맵을 생성하였다. 평가는 5개의 메트릭을 기준으로 수행하였으며, 하나라도 만족하지 않으면 Fail(F), 모두 만족하면 Pass(T)로 판정하였다.

◈ 평가 메트릭
Metric 설명
TR Tile Reflection 시나리오에 등장하지만 게임 월드에 배치되지 않은 타일의 존재 여부
OR Object Reflection 시나리오에 등장하지만 게임 월드에 배치되지 않은 오브젝트의 존재 여부
TA Tile Accessibility 플레이어가 접근할 수 없는 이동 가능 타일의 존재 여부
OP Object Placement 이동 불가 타일 위에 배치된 오브젝트의 존재 여부
OA Object Accessibility 플레이어가 접근할 수 없는 오브젝트의 존재 여부
각 조건에서 생성된 10개의 맵에 대해 5개 메트릭을 평가하여, 모든 메트릭을 만족하면 T (Pass), 하나라도 만족하지 않으면 F (Fail)로 판정하였다. 아래 표의 F Count는 10개 맵 중 Fail 판정을 받은 맵의 수를 나타낸다.
