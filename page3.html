같은 게임, 다른 이야기 — 매 회차마다 다른 시나리오로 진행되는 RPG

Problem 게임 내 고정된 퀘스트와 시나리오로 인한 반복 플레이는 리플레이 가치가 떨어진다.
Approach 매 회차마다 새로운 시나리오를 생성하고, 실제 게임에 연동하여 플레이 가능하도록 구현한다.
Period 2023.08 – 2025.08
Type 졸업 논문 / 연구 프로젝트

Stack C# · Python · Unity · Claude API · OpenAI API · Slack · GitLab

연구 개요 Research Overview

STEP 01
시나리오 대본 생성
⚠️ 실시간으로 LLM을 호출하는 방식은 네트워크 지연으로 인해 NPC 반응 속도에 문제가 있다.
💡 시나리오 작법을 프롬프트에 적용하여 게임 로딩 시점에 시나리오 대본을 생성하는 파이프라인을 설계했다.
생성된 시나리오 대본을 토대로 실제 게임 월드를 구성하는 단계로 확장
STEP 02
게임 월드 구성
⚠️ 게임 월드는 지형, NPC 등 다양한 요소를 고려하는 과정에서 생기는 복잡성 때문에 한 번에 생성하기 어렵다.
💡 월드 생성 파이프라인과 Self-Reflection 기법을 단계적으로 적용하여 플레이 가능한 게임 월드를 구성했다.
시나리오 대본 생성 과정에서 게임의 시스템적 제약 사항을 고려하는 단계로 확장
STEP 03
게임 시스템 연동
⚠️ 대본에서 연출되는 상황에 따라 구현해야 하는 이벤트가 다양하여, 범용적인 시스템 구조 설계가 필요했다.
💡 대화 유형을 분류하여 각종 이벤트, 퀘스트 등을 시스템과 연동하는 과정을 파이프라인에 추가했다.
