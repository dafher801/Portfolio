Snowing 패턴 Curved Projectile

투사체가 단순 직선이 아닌 일정한 곡선 패턴을 그리며 낙하해야 했다. 스플라인 경로를 미리 만들어 따라가게 하는 방법도 있었지만, 곡선의 폭이나 빈도를 바꿀 때마다 경로를 새로 그려야 하는 문제가 있었다.

🧠
판단
Decision
→ 경로 수동 지정 대신 프레임당 회전 각도를 누적하고, 누적값이 임계점에 도달하면 부호를 반전시켜 곡선을 수학적으로 생성
✓
결과
Result
→ 곡선의 폭·빈도를 하나의 값으로 제어하여 경로 수동 지정 없이 원하는 패턴 생성 가능

Bomb 패턴 Area-based Scatter

지정된 영역에 랜덤으로 탄막을 흩뿌려야 했는데, 순수 랜덤만으로는 탄막이 한 곳에 뭉치거나 과도하게 퍼져서 플레이어가 보스의 탄막을 회피하는 행위에 몰입이 방해되는 현상이 발생했다.

🧠
판단
Decision
→ 영역 좌표로 산포 범위를 정의하고, 투사체 생성 시 기존 투사체들과의 최소 간격 검증을 거쳐 통과한 경우에만 배치
✓
결과
Result
→ 탄막이 시각적으로 고르게 분포되어 회피 가능하면서도 긴장감 있는 패턴을 구성
→
두 패턴 모두 "경로나 위치를 직접 찍어주는" 방식 대신 수학적 규칙을 설계하여 런타임에 계산하는 접근을 택했다. 미리 만든 에셋에 의존하지 않으니 파라미터 조정만으로 변형이 자유로웠고, 이 경험은 이후 SnowTale에서 전투 시스템 전체를 데이터 주도로 설계하는 밑바탕이 되었다.
